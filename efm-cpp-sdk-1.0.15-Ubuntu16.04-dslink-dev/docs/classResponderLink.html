Detailed Description

The responder link example demonstrates the EFM SDK API for responder implementations.

Shows node and action creation, stream handling and task scheduling.

Constructor & Destructor Documentation

Constructs the responder link implementation.

Adds a node creation pattern to the responder in order to add default actions and configs to nodes created as children from the /nodes path.

Node creation pattern callback for the '/nodes/*' pattern.

Will be fired for every node created that matches the pattern. Will add default actions and configs to the node. Will add nothing to action nodes, as the pattern will match every node, even actions.

Parameters
pathThe path of the node the pattern matched for.
contextThe context specifies in which operational phase the node was created.

Callback that will be called upon construction of the first level nodes.

Will add an action to the /nodes path. This action could also be added via a node creation pattern callback. The action will add a new child node to the /nodes path.

Parameters
